## Overview

The Launcher IPC module (`src/main/ipc/launcherIpc.js`) handles the core launcher functionality including startup checks, manifest fetching, mod synchronization, and Minecraft launching.

## Handlers

### startup-check

Performs initial startup check to verify Java availability.

<ParamField path="Returns" type="object">
  <Expandable title="properties">
    <ParamField path="javaPath" type="string">
      Resolved path to Java executable
    </ParamField>
    <ParamField path="ok" type="boolean">
      Whether Java is available
    </ParamField>
  </Expandable>
</ParamField>

#### Example

```javascript
// Renderer process
const result = await window.launcher.startupCheck();

if (result.ok) {
  console.log('Java found at:', result.javaPath);
} else {
  console.error('Java not found');
}
```

#### Implementation

```javascript
// src/main/ipc/launcherIpc.js:14-19
ipcMain.handle('startup-check', async () => {
  return {
    javaPath: state.resolvedJavaPath,
    ok: !!state.resolvedJavaPath,
  };
});
```

### fetch-manifest

Fetches the server manifest containing mod lists and game version info. Falls back to cached manifest if server is unreachable.

<ParamField path="Returns" type="object">
  <Expandable title="properties">
    <ParamField path="ok" type="boolean">
      Whether the operation succeeded
    </ParamField>
    <ParamField path="manifest" type="object" optional>
      The manifest data containing profiles and mod lists
    </ParamField>
    <ParamField path="cached" type="boolean" optional>
      True if manifest was loaded from cache due to network error
    </ParamField>
    <ParamField path="error" type="string" optional>
      Error message if operation failed
    </ParamField>
  </Expandable>
</ParamField>

#### Example

```javascript
// Renderer process
const { ok, manifest, cached, error } = await window.launcher.fetchManifest();

if (ok) {
  if (cached) {
    console.warn('Using cached manifest - server unreachable');
  }
  console.log('Manifest:', manifest);
} else {
  console.error('Failed to load manifest:', error);
}
```

#### Implementation

```javascript
// src/main/ipc/launcherIpc.js:21-37
ipcMain.handle('fetch-manifest', async () => {
  try {
    const manifest = await fetchManifest();
    fs.writeFileSync(
      path.join(BASE_DIR, 'cache', 'manifest.json'),
      JSON.stringify(manifest, null, 2)
    );
    return { ok: true, manifest };
  } catch (err) {
    const cachePath = path.join(BASE_DIR, 'cache', 'manifest.json');
    if (fs.existsSync(cachePath)) {
      log(`Could not reach server, using cached manifest: ${err.message}`);
      return { ok: true, manifest: JSON.parse(fs.readFileSync(cachePath, 'utf8')), cached: true };
    }
    return { ok: false, error: err.message };
  }
});
```

### sync-and-launch

Synchronizes mods and launches Minecraft. This is the main launcher operation that:
1. Verifies Java installation
2. Installs Forge if needed
3. Syncs mods from manifest
4. Launches Minecraft

Emits progress events: `status`, `phase`, and `launched`.

<ParamField path="opts" type="object">
  <Expandable title="properties">
    <ParamField path="config" type="object" required>
      User configuration object
      <Expandable title="config properties">
        <ParamField path="selectedProfile" type="string">
          Profile name to launch (defaults to DEFAULT_PROFILE)
        </ParamField>
        <ParamField path="minecraftDir" type="string">
          Minecraft installation directory
        </ParamField>
        <ParamField path="ram" type="number">
          RAM allocation in MB (default: 4096)
        </ParamField>
        <ParamField path="username" type="string">
          Player username (default: 'Player')
        </ParamField>
      </Expandable>
    </ParamField>
    <ParamField path="manifest" type="object" required>
      Manifest object from fetch-manifest
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="Returns" type="object">
  <Expandable title="properties">
    <ParamField path="ok" type="boolean">
      Whether launch succeeded
    </ParamField>
    <ParamField path="pid" type="number" optional>
      Process ID of launched Minecraft instance
    </ParamField>
    <ParamField path="error" type="string" optional>
      Error message if launch failed
    </ParamField>
  </Expandable>
</ParamField>

#### Example

```javascript
// Renderer process
const config = await window.launcher.loadConfig();
const { ok, manifest } = await window.launcher.fetchManifest();

if (ok) {
  // Listen for progress events
  window.launcher.on('status', (msg) => {
    console.log('Status:', msg);
  });

  window.launcher.on('phase', (phase) => {
    console.log('Phase:', phase); // 'java', 'forge', 'mods', 'launch'
  });

  window.launcher.on('launched', ({ pid }) => {
    console.log('Minecraft launched with PID:', pid);
  });

  const result = await window.launcher.syncAndLaunch({ config, manifest });
  
  if (!result.ok) {
    console.error('Launch failed:', result.error);
  }
}
```

#### Implementation

```javascript
// src/main/ipc/launcherIpc.js:39-88
ipcMain.handle('sync-and-launch', async (_, { config, manifest }) => {
  try {
    const profileName = config.selectedProfile || CONFIG.DEFAULT_PROFILE;
    const profileManifest = manifest.profiles?.[profileName] || manifest;

    const mcVersion = profileManifest.minecraftVersion || LATEST_MC_VERSION;
    const forgeVersion = profileManifest.forgeVersion || null;
    const javaInfo = getRequiredJavaVersion(mcVersion);

    const minecraftDir = config.minecraftDir || BASE_DIR;

    send('status', `Verificando Java ${javaInfo.major}...`);
    send('phase', 'java');
    const javaPath = resolveJavaForLaunch(javaInfo.major);

    if (forgeVersion) {
      send('status', `Verificando Forge ${forgeVersion}...`);
      send('phase', 'forge');
      await installForge(javaPath, minecraftDir, mcVersion, forgeVersion);
    }

    send('status', 'Sincronizando mods...');
    send('phase', 'mods');
    const updatedCount = await syncMods(profileManifest, minecraftDir);

    if (updatedCount > 0) {
      log(`${updatedCount} mod(s) atualizado(s).`);
    } else {
      log('Todos os mods estão atualizados!');
    }

    send('status', 'Iniciando Minecraft...');
    send('phase', 'launch');

    const pid = await launchMinecraft({
      javaPath,
      minecraftDir,
      ram: config.ram || 4096,
      username: config.username || 'Player',
      mcVersion,
      forgeVersion,
    });

    send('launched', { pid });
    return { ok: true, pid };
  } catch (err) {
    log(`ERRO: ${err.message}`);
    return { ok: false, error: err.message };
  }
});
```

## Event Timeline

During `sync-and-launch`, events are emitted in this order:

1. **Phase: java** - `status: "Verificando Java {version}..."`
2. **Phase: forge** - `status: "Verificando Forge {version}..."` (if Forge is required)
3. **Phase: mods** - `status: "Sincronizando mods..."`
4. **Phase: launch** - `status: "Iniciando Minecraft..."`
5. **Event: launched** - `{ pid: number }`

## Related Services

These handlers delegate to the following services:

- `manifestService.js` - Fetches manifest from remote server
- `modSyncService.js` - Downloads and verifies mod files
- `minecraftLauncher.js` - Constructs and executes Minecraft launch command
- `forgeInstaller.js` - Downloads and installs Forge if needed
