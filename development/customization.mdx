## Overview

The Umucraft Launcher is designed to be easily customized for different Minecraft servers. This guide covers all customization points including branding, server configuration, and behavior modifications.

## Manifest URL Configuration

The most critical configuration is the manifest URL, which determines where the launcher downloads mod pack information.

<Steps>
  <Step title="Open paths.js">
    Navigate to `src/main/utils/paths.js`
  </Step>

  <Step title="Update the MANIFEST_URL">
    Modify the `CONFIG` object:

    ```javascript src/main/utils/paths.js
    const CONFIG = {
      MANIFEST_URL: 'https://your-url.com/manifest.json',
      DEFAULT_PROFILE: 'Default',
    };
    ```

    ### Hosting Options

    **Dropbox (Recommended for beginners)**
    ```javascript
    MANIFEST_URL: 'https://www.dropbox.com/scl/fi/XXX/manifest.json?rlkey=YYY&dl=1'
    ```
    <Note>Change `?dl=0` to `?dl=1` in Dropbox share links</Note>

    **GitHub Raw**
    ```javascript
    MANIFEST_URL: 'https://raw.githubusercontent.com/user/repo/main/manifest.json'
    ```

    **Self-Hosted**
    ```javascript
    MANIFEST_URL: 'https://yourserver.com/launcher/manifest.json'
    ```
  </Step>

  <Step title="Update default profile (optional)">
    If your manifest uses a different profile name:

    ```javascript
    const CONFIG = {
      MANIFEST_URL: 'https://your-url.com/manifest.json',
      DEFAULT_PROFILE: 'YourProfileName',  // Must match manifest.json
    };
    ```
  </Step>
</Steps>

## Server List Configuration

Configure which Minecraft servers appear in the Servers tab with ping functionality.

<Steps>
  <Step title="Open servers.js">
    Navigate to `src/renderer/data/servers.js`
  </Step>

  <Step title="Edit the SERVERS array">
    ```javascript src/renderer/data/servers.js
    export const SERVERS = [
      { 
        name: 'My Modded Server', 
        host: 'mc.myserver.com', 
        port: 25565 
      },
      { 
        name: 'Creative Server', 
        host: 'creative.myserver.com', 
        port: 25566 
      },
      { 
        name: 'Vanilla Server', 
        host: 'vanilla.myserver.com', 
        port: 25565 
      },
    ];
    ```

    ### Server Object Properties

    | Property | Type | Description |
    |----------|------|-------------|
    | `name` | string | Display name shown in the launcher |
    | `host` | string | Server hostname or IP address |
    | `port` | number | Server port (usually 25565) |
  </Step>

  <Step title="Test server connectivity">
    The launcher will ping servers automatically when the Servers tab is opened. Ensure:
    - Servers are online
    - Ports are not firewalled
    - Hostnames resolve correctly
  </Step>
</Steps>

<Steps>

  <Step title="Customize color variables">

      --bg-dark:    #0d1117;    /* Main background */
      --bg-card:    #161b22;    /* Card/panel background */
      --bg-hover:   #1c2333;    /* Hover state background */
      
      /* Border Colors */
      --border:     #30363d;    /* Standard borders */

      --accent-glow:#4ade8022;  /* Glow effect */
      
      /* Feature Colors */
      --forge:      #f97316;    /* Forge orange */
      --java-col:   #60a5fa;    /* Java blue */
      --discord:    #5865F2;    /* Discord blurple */
      
      /* Text Colors */
      --text:       #e6edf3;    /* Primary text */
      --text-dim:   #8b949e;    /* Secondary text */

## Java Version Requirements

Configure which Java versions are required for different Minecraft versions.

<Steps>
  <Step title="Open paths.js">
    Navigate to `src/main/utils/paths.js`
  </Step>

  <Step title="Update JAVA_VERSIONS mapping">
    ```javascript src/main/utils/paths.js
    const JAVA_VERSIONS = {
      '1.17': { version: '17', major: 17 },
      '1.18': { version: '17', major: 17 },
      '1.19': { version: '17', major: 17 },
      '1.20': { version: '21', major: 21 },
      '1.21': { version: '21', major: 21 },
    };
    ```

    The launcher uses this mapping to:
    - Determine required Java version during bootstrap
    - Download the correct Java distribution
    - Validate existing Java installations
  </Step>
</Steps>

## Application Branding

### Product Name and App ID

Update the application identity in `package.json`:

```json package.json
{
  "name": "YourLauncherName",
  "version": "1.0.0",
  "description": "Custom Minecraft Launcher for Your Server",
  "build": {
    "appId": "com.yourserver.launcher",
    "productName": "Your Server Launcher"
  }
}
```

### Application Icons

Replace the default icons in `src/assets/`:

- **Windows**: `icon.ico` (256x256 pixels)
- **Linux**: `icon.png` (512x512 pixels)
- **macOS**: `icon.icns` (1024x1024 pixels)

<Tip>
  Use tools like [iConvert Icons](https://iconverticons.com/) to convert between icon formats.
</Tip>

## Data Directory Location

The launcher stores data in the user's home directory. To change this:

```javascript src/main/utils/paths.js
const BASE_DIR = path.join(os.homedir(), '.YourLauncherName');
```

### Directory Structure

The launcher creates these subdirectories:

```javascript src/main/utils/paths.js
function ensureDirectories() {
  const dirs = [
    'java',       // Bundled Java runtime
    'versions',   // Minecraft client jars
    'mods',       // Synchronized mods
    'modpacks',   // Mod packs
    'config',     // Mod configurations
    'assets',     // Minecraft assets
    'cache',      // Cached manifest
    'libraries',  // Minecraft/Forge libraries
    'logs'        // Bootstrap logs
  ];
  dirs.forEach(d => fs.mkdirSync(path.join(BASE_DIR, d), { recursive: true }));
}
```

Set the default RAM amount in the configuration:

```javascript src/main/services/configService.js
const DEFAULT_CONFIG = {
  username: '',
  ram: '4096',           // Default: 4GB
  gameDirectory: BASE_DIR,
  javaPath: '',
  selectedProfile: CONFIG.DEFAULT_PROFILE,
};
```

## Testing Customizations

After making customizations:

<Steps>
  <Step title="Clear cached data">
    Remove the data directory to test fresh installations:

    <CodeGroup>
    ```bash Windows
    rmdir /s %USERPROFILE%\.UmuCraft
    ```

    ```bash Linux/macOS
    rm -rf ~/.UmuCraft
    ```
    </CodeGroup>
  </Step>

  <Step title="Test in development mode">
    ```bash
    npm run dev
    ```
  </Step>

  <Step title="Build and test installer">
    ```bash
    npm run build
    ```
    
    Install the generated package and verify all customizations appear correctly.
  </Step>
</Steps>

## Checklist for Custom Deployments

Before distributing your customized launcher:

- [ ] Update manifest URL in `paths.js`
- [ ] Configure server list in `servers.js`
- [ ] Set Discord invite in `discord.js`
- [ ] Customize colors in `variables.css`
- [ ] Replace application icons in `assets/`
- [ ] Update product name and app ID in `package.json`
- [ ] Test all features in development mode
- [ ] Build and test installer on clean system
- [ ] Verify manifest downloads correctly
- [ ] Confirm server pinging works
- [ ] Test game launch with mod synchronization
